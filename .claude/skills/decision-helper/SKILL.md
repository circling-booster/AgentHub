# Decision Helper

ADHD 특성과 선택장애를 고려한 의사결정 지원 도구.

## 핵심 원칙

| 원칙 | 이유 |
|------|------|
| **4개 이하 선택지** | 선택지가 많으면 결정 마비 |
| **시각적 구분** | 🔴🟡🟢 긴급도로 주의 집중 |
| **문서로 고정** | 휘발성 생각을 즉시 기록 |
| **추천 명시** | "(추천)" 표시로 결정 부담 감소 |

## 워크플로우

```
1. 상황 파악 → 2. 문서 생성 → 3. 검토 질문 → 4. 결정 → 5. 기록
```

### 1단계: 상황 파악

사용자 발언에서 핵심 추출:

```
[사용자가 말한 것]
- 핵심 문제: ???
- 관련 맥락: ???
- 제약 조건: ???
- 감지된 감정: (불안/혼란/급함 등)
```

### 2단계: 결정 문서 생성

`docs/decisions/` 폴더에 마크다운 파일 생성:

**파일명 규칙:** `YYYY-MM-DD-{주제}.md`

**필수 구조:**

```markdown
# 🎯 [결정 주제]

**상태:** 🟡 검토 중 | **긴급도:** 🔴높음/🟡보통/🟢낮음 | **날짜:** YYYY-MM-DD

---

## 📌 한 줄 요약
> [상황을 한 문장으로]

## 🔍 현재 상황
- 핵심 문제:
- 배경:
- 제약:

## ⚖️ 선택지 비교

| 선택지 | 장점 | 단점 | 비용/노력 |
|--------|------|------|-----------|
| **A: [이름]** (추천) | - | - | 🟢낮음 |
| **B: [이름]** | - | - | 🟡보통 |
| **C: [이름]** | - | - | 🔴높음 |

## ❓ 결정 전 체크리스트

- [ ] [질문1]?
- [ ] [질문2]?
- [ ] [질문3]?

## 🎬 선택하기

> 👆 위 체크리스트 확인 후, 아래에서 선택:
> - [ ] **A 선택** - [한 줄 설명]
> - [ ] **B 선택** - [한 줄 설명]
> - [ ] **C 선택** - [한 줄 설명]

---
## 📝 결정 기록 (결정 후 작성)

**최종 선택:**
**선택 이유:**
**예상 결과:**
**회고 날짜:** (1주 후 / 1달 후)
```

### 3단계: 체크리스트 질문

문서 생성 후, 사용자에게 핵심 질문을 던진다:

- 최대 10개 질문
- 예/아니오로 답할 수 있는 형태 우선
- AskUserQuestion 도구 사용 권장

**질문 유형:**
- "이게 맞나요?" (확인)
- "이것과 저것 중 뭐가 더 중요해요?" (우선순위)
- "빠진 게 있나요?" (완전성)

### 4단계: 결정 지원

사용자가 결정하면:
1. 문서의 "결정 기록" 섹션 업데이트
2. 상태를 "✅ 결정됨"으로 변경
3. 다음 액션 아이템 제시

### 5단계: 결정 로그

모든 결정은 추적 가능하게:

```
docs/decisions/
├── 2026-01-28-tech-stack.md     ✅ 결정됨
├── 2026-01-29-architecture.md   🟡 검토 중
└── index.md                      # 결정 목록
```

## 긴급도 가이드

| 표시 | 의미 | 기준 |
|------|------|------|
| 🔴 | 지금 당장 | 오늘 안에 결정 필요 |
| 🟡 | 이번 주 | 급하진 않지만 곧 필요 |
| 🟢 | 여유 있음 | 나중에 해도 됨 |

## 추천 표시 규칙

Claude가 추천을 할 때:

```markdown
| **A: 빠른 방법** (추천) | 간단함 | 확장성 낮음 | 🟢 |
```

**추천 기준:**
- 리스크 대비 효과가 가장 좋은 옵션
- 되돌리기 쉬운 옵션 (실패해도 복구 가능)
- 현재 상황에 가장 적합한 옵션

## 자주 쓰는 패턴

```markdown
## ⚖️ 기술 스택 비교

|------|----------|--------|-----------|
| **React** (추천) | 🟡 | 풍부 | 레퍼런스 많음 |
| **Vue** | 🟢 | 보통 | 간단한 프로젝트에 적합 |
| **Svelte** | 🟡 | 성장중 | 새로운 시도 원하면 |
```

### 우선순위 결정

```markdown
## ⚖️ 뭘 먼저 할까?

| 작업 | 긴급 | 중요 | 추천 순서 |
|------|------|------|-----------|
| 버그 수정 | 🔴 | 🟡 | 1순위 |
| 새 기능 | 🟢 | 🔴 | 2순위 |
| 리팩토링 | 🟢 | 🟢 | 3순위 |
```

## 사용 예시

**사용자:** "MCP랑 A2A 중에 뭘 먼저 해야 할지 모르겠어. MCP는 도구 연결이고 A2A는 에이전트 간 통신인데... 둘 다 필요한 것 같은데 시간이 없어."

**Claude 응답:**

1. `docs/decisions/2026-01-28-mcp-vs-a2a.md` 생성
2. 상황 요약 + 비교표 작성
3. 체크리스트 질문:
   - "지금 당장 연동할 외부 도구가 있나요?"
   - "다른 에이전트와 통신이 필요한 시나리오가 있나요?"
4. 추천과 함께 선택지 제시
