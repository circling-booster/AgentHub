# 코드 리뷰

대상: `$ARGUMENTS` (없으면 최근 변경사항)

## 리뷰 관점

### 1. 보안 (Security)

- SQL Injection 취약점
- XSS (Cross-Site Scripting)
- 하드코딩된 자격 증명
- 안전하지 않은 역직렬화
- 경로 탐색 취약점
- 민감 정보 노출

### 2. 성능 (Performance)

- N+1 쿼리 문제
- 불필요한 반복문
- 메모리 누수 가능성
- 비효율적인 알고리즘
- 캐싱 기회

### 3. 가독성 (Readability)

- 명확한 변수/함수명
- 적절한 주석
- 일관된 코드 스타일
- 함수 길이 및 복잡도
- 매직 넘버 사용

### 4. 에러 핸들링

- 예외 처리 누락
- 불명확한 에러 메시지
- 리소스 정리 (finally, with)
- 에러 로깅

### 5. 테스트 가능성

- 의존성 주입
- 모킹 가능 여부
- 단위 테스트 작성 용이성

## 출력 형식

```markdown
# 코드 리뷰 결과

## 요약

- 심각도 높은 이슈: N개

## 이슈 목록

### 🟡 주의 (Warning)
...

### 🟢 제안 (Suggestion)
...

## 칭찬할 점
- 잘 구현된 부분 언급

## 권장 수정 코드
[수정 코드 제시]
```

## 리뷰 깊이

- 단일 파일: 상세 리뷰
- 디렉토리: 주요 이슈 중심
- 전체 변경: 차이점 중심 리뷰
