# 코드 설명

대상: `$ARGUMENTS`

## 설명 구조

코드를 설명할 때 다음 구조를 따릅니다:

### 1. 비유로 시작

코드를 일상적인 것에 비유하여 직관적으로 이해할 수 있게 합니다.

예시:
- "이 함수는 우체국 분류 시스템과 같습니다..."
- "이 클래스는 레스토랑 주방장처럼 동작합니다..."

### 2. ASCII 다이어그램

흐름, 구조, 관계를 시각적으로 표현합니다.

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│  Input  │────▶│ Process │────▶│ Output  │
└─────────┘     └─────────┘     └─────────┘
```

### 3. 단계별 설명

코드의 각 부분이 무엇을 하는지 순서대로 설명합니다.

```
1. 입력 검증: 데이터가 올바른 형식인지 확인
2. 데이터 변환: 내부 형식으로 변환
3. 핵심 로직: 실제 처리 수행
4. 결과 반환: 처리 결과를 반환
```

### 4. 함정과 주의사항

흔히 발생하는 실수나 오해할 수 있는 부분을 강조합니다.

- "주의: 이 함수는 None을 반환할 수 있습니다"
- "함정: 인덱스가 0이 아닌 1부터 시작합니다"

## 설명 깊이

- 파일 전체: 전체 구조와 주요 컴포넌트 관계
- 클래스: 책임, 속성, 메서드 역할
- 함수: 입력, 처리, 출력, 부작용
- 코드 블록: 세부 로직 설명

## 출력 형식

```markdown
# [파일/함수명] 설명

[간결한 설명]

[일상적 비유]

## 구조
[ASCII 다이어그램]

## 상세 설명
[단계별 설명]

## 주의사항
[함정 및 팁]
```
