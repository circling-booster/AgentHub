# 프로젝트 구조 분석

## 분석 항목

### 1. 디렉토리 구조

프로젝트의 주요 디렉토리를 트리 형태로 표시합니다.

```
프로젝트/
├── src/           # 소스 코드
├── tests/         # 테스트
├── docs/          # 문서
└── ...
```

### 2. 기술 스택

- 언어 및 버전
- 주요 프레임워크
- 의존성 (requirements.txt, package.json 등)

### 3. 진입점

- 메인 파일 위치
- 실행 방법
- 설정 파일

### 4. 아키텍처 패턴

- 사용된 디자인 패턴
- 레이어 구조
- 모듈 간 의존 관계

### 5. 주요 컴포넌트

각 주요 디렉토리/모듈의 역할 설명

## 출력 형식

```markdown
# 프로젝트 개요: [프로젝트명]

## 기술 스택
- 언어: Python 3.10+
- 프레임워크: FastAPI
- ...

## 디렉토리 구조
[트리 다이어그램]

## 아키텍처
[ASCII 다이어그램]

## 주요 파일
| 파일 | 역할 |
|------|------|
| main.py | 진입점 |
| ... | ... |

## 실행 방법
[명령어]

## 다음 단계
- 추천 탐색 경로
- 주요 코드 위치
```

## 탐색 전략

1. 루트 디렉토리 파일 확인 (README, pyproject.toml 등)
2. 소스 코드 디렉토리 구조 파악
3. 설정 파일 분석
4. 주요 모듈 진입점 확인
