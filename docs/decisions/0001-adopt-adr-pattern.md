# ADR-0001: ADR 패턴 도입

## 상태

승인됨

## 날짜

2026-01-28

## 컨텍스트

AgentHub 프로젝트 구현 중 다음과 같은 문제가 예상됩니다:

1. 원래 계획과 다른 방식으로 구현했을 때 기록이 남지 않음
2. 왜 특정 기술이나 패턴을 선택했는지 나중에 알기 어려움
3. 팀원(미래의 나 포함)이 과거 결정의 맥락을 이해하기 어려움

## 결정

**Architecture Decision Records (ADR) 패턴을 도입합니다.**

- 위치: `docs/decisions/`
- 명명 규칙: `NNNN-kebab-case-title.md`
- 템플릿: `TEMPLATE.md` 사용

### 작성 시점

- 원래 계획과 다른 구현을 선택했을 때
- Domain Layer에 변경이 발생했을 때
- 새로운 외부 라이브러리를 도입했을 때
- 아키텍처 패턴을 변경했을 때

## 대안

### 대안 1: 로드맵 내 인라인 기록
- 장점: 단일 문서, 간단함
- 단점: 문서가 길어지고, 변경 추적 어려움

### 대안 2: 커밋 메시지에만 기록
- 장점: 별도 문서 불필요
- 단점: 검색 어려움, 맥락 부족

### 대안 3: 기록하지 않음
- 장점: 오버헤드 없음
- 단점: 나중에 왜 그렇게 했는지 모름

## 결과

### 긍정적 영향
- 중요한 결정의 맥락이 보존됨
- 새로운 참여자가 프로젝트를 이해하기 쉬워짐
- 과거 결정을 재검토할 때 참고 가능

### 부정적 영향 / 트레이드오프
- ADR 작성에 시간 소요 (경미함)
- 모든 결정에 ADR을 작성하면 오버헤드 발생 → 주요 결정에만 작성
