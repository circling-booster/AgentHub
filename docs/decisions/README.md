# Architecture Decision Records (ADR)

> 프로젝트의 중요한 아키텍처 결정을 기록하는 폴더

## ADR이란?

ADR(Architecture Decision Record)은 소프트웨어 아키텍처에 관한 중요한 결정을 문서화하는 방법입니다. 각 ADR은 단일 결정과 그 맥락, 결과를 기록합니다.

## 언제 ADR을 작성하는가?

- 원래 계획과 다른 방식으로 구현했을 때
- 아키텍처에 영향을 주는 기술 선택을 했을 때
- 향후 참고가 필요한 중요한 결정을 내렸을 때
- 대안들 중 하나를 선택해야 했을 때

## 상태 정의

- **제안됨**: 검토 중
- **승인됨**: 채택되어 적용 중
- **폐기됨**: 더 이상 유효하지 않음
- **대체됨**: 다른 ADR로 대체됨
